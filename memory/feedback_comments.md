---
name: Sparse comments
description: Only add comments to genuinely complex code; omit them elsewhere
type: feedback
---

Use comments sparingly — only when the code is complex enough that the WHY is non-obvious without them. Skip comments on straightforward code.

**Why:** User preference; self-explanatory code doesn't need narration.

**How to apply:** Default to no comments. Add one only when there's a hidden constraint, subtle invariant, or behaviour that would surprise a reader.
